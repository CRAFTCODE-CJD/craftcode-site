## 13.5 i18n ENGINE

Реализовать:

```ts id="lj1i9l"
let currentLang = "en";

export function setLang(lang: "en" | "ru") {
  currentLang = lang;
  localStorage.setItem("lang", lang);
}

export function getLang(): string {
  return localStorage.getItem("lang") || "en";
}

export function t(key: string): string {
  return translations[currentLang][key] || key;
}
```

# 14. AUTO i18n ДЛЯ MDX (ГЕНЕРАЦИЯ КЛЮЧЕЙ И ПЕРЕВОДОВ)

## 14.1 Цель

Автоматизировать локализацию контента:

* извлекать текст из MDX
* генерировать i18n ключи
* автоматически формировать `en.json` / `ru.json`
* заменять текст в MDX на `t("key")`

❗ Система должна работать без ручного дублирования контента.

---

## 14.2 ОБЩИЙ PIPELINE

```id="q1z9mw"
MDX → AST парсинг → извлечение текста → генерация ключей →
обновление translations → замена текста → финальный MDX
```

---

## 14.3 СТРУКТУРА

```id="d5yq8r"
scripts/
  extract-i18n.ts

src/i18n/
  translations/
    en.json
    ru.json
```

---

## 14.4 ЧТО НУЖНО ИЗВЛЕКАТЬ

Скрипт обязан извлекать:

* текстовые ноды
* строки внутри компонентов (`<Hero title="..." />`)
* markdown текст
* содержимое `<Step>`, `<Block>` и т.д.

❗ Игнорировать:

* code blocks
* inline code
* технические значения (id, src, пути)

---

## 14.5 ГЕНЕРАЦИЯ КЛЮЧЕЙ

### Требование

Ключи должны быть:

* читаемыми
* стабильными
* привязанными к файлу

### Пример

```id="7b4q2p"
plugin.replicatedCursor.hero.title
plugin.replicatedCursor.description
plugin.replicatedCursor.steps.addComponent
```

❗ Запрещено:

```id="0i3kts"
text1
text2
random_key
```

---

## 14.6 ПРАВИЛА ИМЕНОВАНИЯ

```id="9qj2xm"
[domain].[entity].[section].[element]
```

Пример:

```id="m8y5ru"
plugin.cursor.step.enableReplication
```

---

## 14.8 AST ПАРСИНГ (ОБЯЗАТЕЛЬНО)

Использовать:

* `@mdx-js/mdx`
* `remark`
* `unist-util-visit`

❗ Regex-решения запрещены для production.

---

## 14.9 ЛОГИКА СКРИПТА

1. Прочитать MDX файл
2. Построить AST
3. Найти текстовые ноды
4. Сгенерировать ключ
5. Добавить в `en.json`
6. Если ключ новый:

   * добавить в `ru.json` пустое значение
7. Заменить текст в AST на `t(key)`
8. Сохранить файл

---

## 14.10 ОБНОВЛЕНИЕ ПЕРЕВОДОВ

### Поведение

* существующие ключи не перезаписывать
* новые добавлять
* удалённые помечать (optional)
