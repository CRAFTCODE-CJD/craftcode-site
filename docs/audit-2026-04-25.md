**Note on SEO score (66):** Cloudflare Pages automatically injects `x-robots-tag: noindex` on all preview (non-production) deployments. This causes Lighthouse SEO to fail the crawlability audit. On production (`craftcode-site.pages.dev` or custom domain) this header is absent and SEO will score 100. This is expected and not a code issue.

## Deferred / known issues

- 7 moderate vulnerabilities (astro <6.1.6 XSS, yaml stack-overflow chain) — fix requires major bumps; defer to next cycle.
- i18n race window: Base.astro calls `initI18n()` without await; for ~50–200ms, `t()` returns null. Visual fallback is correct (SSR text retained). Future fix: top-level await in module script.
- Lighthouse performance on home page is 60 (preview) — LCP 8.8s driven by KaplayPlayground canvas even after lazy loading. Improvement options: reduce canvas resolution, skip animation in prefers-reduced-motion, or replace with static screenshot until user interacts.
- Lighthouse CI workflow uses `temporary-public-storage`; LHCI_GITHUB_APP_TOKEN not yet set (would enable PR comments).
- Husky lint-staged runs placeholder commands; future task should add real Prettier/typecheck.
- `Cross-Origin-Embedder-Policy` header not set — needed to reach A+ on securityheaders.com (requires all embedded resources to opt in via COEP, which conflicts with YouTube iframes).

## Recommendations for next audit (~6 mo)

- Re-run Lighthouse after major content additions
- Bump major-locked deps (astro, yaml chain) deliberately, run full QA
- Consider hstspreload.org submission once domain is finalized
- Add `Cross-Origin-Embedder-Policy: credentialless` to reach A+ on securityheaders.com (verify YouTube embeds still work)
- Add Cyrillic-subset font preloads if RU traffic dominates
- Investigate LCP optimisation: static poster image for KaplayPlayground, lazy canvas init
