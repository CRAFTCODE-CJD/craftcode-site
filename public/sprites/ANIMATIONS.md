# CraftCode Companions — Sprite Animation Reference

> **Для AI-ассистентов / игровых движков:** этот файл описывает каждый кадр в
> `craft.png` и `code.png`, его семантику, и подсказывает когда какую клипу
> проигрывать. Листы — источник истины; этот документ — их «контракт».
> Источник генерации: [`Scripts/generate_companions.py`](../../../Scripts/generate_companions.py).

---

## 1. Два персонажа

Оба — маленькие квадратные роботы-компаньоны бренда CraftCode. Различаются
только **силуэтом** и **цветом**; разделение жёсткое — перепутать нельзя.

| Персонаж | Силуэт | Основной цвет | Роль |
|---|---|---|---|
| **CRAFT** | Высокий прямоугольник (~10×14 внутри 32×32) | Коралловый / тёплый оранжевый | Художник / строитель / контент-криейтор. Специфика: может размахивать **молотком** (`hammer_*`). |
| **CODE** | Широкий короткий прямоугольник (~14×10 внутри 32×32), 4 коротких ножки | Бирюзовый / минтовый (BMO-отсылка) | Инженер / программист. Специфика: может показать **прогресс-бар** (`bar_*`) над головой. |

**Не используй** CRAFT для «кодерских» сцен и наоборот — пара задумана как
коллеги с разными ролями, не как клоны.

---

## 2. Формат листов

Оба листа идентичны по разметке:

- Файл: `craft.png`, `code.png` (в той же папке, что и этот MD)
- Размер кадра: **32 × 32 px**
- Раскладка: **one row × 92 frames** (2944 × 32 px суммарно)
- Pivot каждого кадра: **bottom-center**, нижняя кромка стопы на строке `y = 26` для CRAFT и `y = 24` для CODE (стабильный baseline — никогда не «уплывает»)
- Нарезка: `frame_i = sheet.crop(i*32, 0, (i+1)*32, 32)`

Любой importer (UE `PaperFlipbook`, Godot `SpriteFrames`, Unity `SpriteSheet`
slicer, LÖVE `quad`, raw canvas) работает с этим из коробки.

### 3.5 Thought / Ideation — **«достаёт ноутбук»** (индексы 60–66)

Это **главная макро-анимация** бренда — персонаж достаёт ноутбук из
«хаммерспейса» (как BMO), ставит его на землю, и начинает печатать.
Источник: `Ref/frames_code/contact_sheet.png`.

| idx | name | поза | роль в цепочке |
|---:|---|---|---|
| 60 | `idea_a` | Стоит, маленький ноутбук высовывается **справа-сверху** от головы по серой леске | «ищу / сейчас достану» |
| 61 | `idea_b` | Правая рука вытягивается вправо, ноутбук висит на серой рукояти | «достал, несу» |
| 62 | `think_a` | Стоит, рука вытянута вправо, ноутбук **стоит на земле справа**, экран-паттерн A | печатает, beat 1 |
| 63 | `think_b` | То же, экран-паттерн B | печатает, beat 2 (чередуется с 62) |
| 64 | `bubble_sm` | Idle + маленькая точка-пузырь над головой | начинает думать |
| 65 | `bubble_lg` | Idle + полный мысленный пузырь с «…» | задумался надолго |
| 66 | `flash` | Idle + яркая жёлтая лампочка с лучами над головой, рот-улыбка | «эврика!» / идея |

**Каноническая цепочка «бот открыл ноут и что-то пишет»:**
```
idle_a → bubble_sm → bubble_lg → flash → idea_a → idea_b → think_a ↔ think_b (loop)
```
FPS: 6-8 для идейной фазы (60-61), 10-12 для печати (62-63).

Внутри генератора кадры — это ASCII-матрицы 32×32 где каждый символ — цвет.

## 5. Silhouette-правила (чтобы не перепутать персонажей)

Если другая нейросеть **генерирует** или **правит** кадры:

**CRAFT (высокий):**
- Тело: колонки 11-21 (10 wide), строки 9-22 (14 tall)
- Руки торчат на строках 16-18, шире тела на 1 колонку с каждой стороны (колонки 10-22)
- Ноги: 2 штуки, в колонках 12-14 и 17-19, строки 23-26
- Глаза: 2×2 pix, в колонках 13-14 и 17-18, строки 11-12
- **Никогда** не рисуй CRAFT с 4 ногами — это сломает брендинг

**CODE (низкий широкий):**
- Тело: колонки 9-22 (14 wide), строки 11-20 (10 tall)
- Руки: на строках 15-17, шире тела на 1 колонку (колонки 7-24)
- Ноги: 2 штуки в текущей версии, в колонках 10-12 и 19-21, строки 21-24
  - _(Заметка: референс `contact_sheet.png` показывает 4 ноги — это допустимая будущая итерация, но текущие листы используют 2 для консистентности с существующими кадрами.)_
- Глаза: 2×2 pix, в колонках 11-12 и 19-20, строки 13-14
- **Никогда** не рисуй CODE выше чем широким — это сломает брендинг

Базовый силуэт сохраняется во **всех** кадрах (идл, ходьба, прыжок, реакции).
Меняется только: положение рук, положение ног, форма глаз, форма рта, и
опциональные оверлеи (пузырь, лампочка, ноутбук, молоток, прогресс-бар).

## 6. Event → Animation mapping (шпаргалка для game AI)

| Игровое событие | Клип | Длительность |
|---|---|---|
| Idle > 0.5s | `idle_a → idle_b` loop | ∞ |
| Idle > 3s | добавь `blink` раз в 3-5s | single frame |
| Idle > 30s | переключи на `dance_*` loop | до следующего события |
| Idle > 60s | `sleep → sleep_1z → sleep_3z` loop | ∞ |
| Idle > 30s + «размышления» | `bubble_sm → bubble_lg → flash → idea_a → idea_b → think_a↔b` loop | ∞ |
| Движение (AI wander / возврат) | walk 6 кадров loop | по длине пути |
| Залез на полку | climb 2 кадра | однократно |

## 7. Технические подсказки импортёрам

### UE (Unreal Paper2D / PaperFlipbook)
```
Source texture: craft.png (или code.png)
Sprite type: ByteWidth=32, ByteHeight=32, Columns=92, Rows=1
Extract → каждому sprite присвой имя из FRAME_ORDER
Flipbook: сгруппируй по клипам (idle={0,1}, walk={5,6,7,8,9,10}, ...)
```

### Godot 4 (SpriteFrames / AnimatedSprite2D)
```gdscript
var frames = SpriteFrames.new()
var tex = load("res://sprites/code.png")
var atlas = AtlasTexture.new(); atlas.atlas = tex
for i in 92:
    var a = atlas.duplicate(); a.region = Rect2(i * 32, 0, 32, 32)
    frames.add_frame(FRAME_NAMES[i], a)
frames.add_animation("walk")
for idx in [5, 6, 7, 8, 9, 10]:
    frames.add_frame("walk", frames.get_frame_texture("all", idx))
frames.set_animation_speed("walk", 10.0)
frames.set_animation_loop("walk", true)
```

### Unity (SpriteRenderer + AnimationClip)
- Import `code.png` → Sprite Mode: Multiple → Sprite Editor → Slice → Grid by Cell Size 32×32
- Создай AnimationClip'ы по индексам из таблицы, Sample Rate = 10-12 для walk, 8 для jump, 2 для idle.

### Web (canvas / CSS)
```css
.bot {
  width: 32px; height: 32px;
  background: url('code.png') 0 0 / 2944px 32px no-repeat;
  image-rendering: pixelated;
}
.bot.walking { animation: walk 0.5s steps(6) infinite; }
@keyframes walk {
  from { background-position: -160px 0; }   /* walk_a = idx 5 */
  to   { background-position: -352px 0; }   /* walk_f = idx 10, end */
}
```

## 8. Что НЕ нужно делать

- ❌ Не миксуй силуэты (CRAFT-голова на CODE-теле и т.п.)
- ❌ Не запускай CRAFT-only клипы (`hammer_*`) на CODE — увидишь idle-заглушку
- ❌ Не запускай CODE-only клипы (`bar_*`) на CRAFT — то же самое
- ❌ Не лупь одноразовые клипы (`jump_*`, `hit_squash`, `facepalm_palm`) — смотрится сломанно
- ❌ Не рендеруй кадры с нативным anti-aliasing или фильтрами — это pixel art, нужен `nearest-neighbor`
- ❌ Не переименовывай кадры — индексы и имена используются в `FRAME_ORDER` как контракт
